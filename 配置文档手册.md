# 配置文档手册

本文档基于当前实现说明项目配置方式、配置项语义、默认值与生效位置。

## 1. 配置入口

- 运行入口是 `app.py`，启动时会读取项目根目录的 `runtime_config.json`。
- `runtime_config.json` 支持两种结构：
  - 推荐：`{"env": {...}}`
  - 兼容：`{"KEY":"VALUE", ...}`
- 配置文件缺失会直接报错退出。

示例：

```json
{
  "env": {
    "WORLD_JSON": "World_SpaceWerewolf.json",
    "VALIDATION_MODE": "fast",
    "USE_LLM": "1",
    "MAX_TICKS": "20",
    "LOG_LEVEL": "info",
    "LOG_CATEGORIES": "system,interaction,checkpoint",
    "CHECKPOINT_EVERY_TICK": "1",
    "CHECKPOINT_INCLUDE_LOGS": "1"
  }
}
```

## 2. 全量配置项（按当前代码）

| 配置项 | 默认值 | 生效位置 | 说明 |
|---|---:|---|---|
| WORLD_JSON | `""` | `app.py` | 指定加载 `Data/<world_file>.json`，空则使用默认 `World.json` |
| VALIDATION_MODE | `"fast"` | `app.py` + `newserver/data/validator.py` | 启动校验模式：`off/fast/strict` |
| DEMO_DURATION_TEST | `"0"` | `app.py` | 内置持续任务演示分支开关 |
| USE_LLM | `"0"` | `app.py` | `1` 使用 LLM 策略，否则使用简单策略 |
| MAX_TICKS | `""` | `app.py` | 空时自动取默认（LLM=15，非LLM=65） |
| CHECKPOINT_EVERY_TICK | `"1"` | `app.py` + `newserver/sim/manager.py` | 是否每 tick 保存 checkpoint |
| CHECKPOINT_INCLUDE_LOGS | `"1"` | `app.py` + `newserver/sim/manager.py` | checkpoint 是否包含事件/交互日志 |
| CHECKPOINT_DIR | `checkpoints/<world_name or default>` | `app.py` + `newserver/sim/manager.py` | checkpoint 输出目录 |
| CHECKPOINT_RESTORE_FILE | `""` | `app.py` + `newserver/data/checkpoint.py` | 从指定 checkpoint 文件恢复 |
| CHECKPOINT_RESTORE_DIR | `""` | `app.py` + `newserver/data/checkpoint.py` | 未指定文件时，从目录中选最新 `tick_*.json` 恢复 |
| LOG_LEVEL | `"info"` | `app.py` + `newserver/log_manager.py` | 日志级别阈值：`trace/debug/info/warn/error` |
| LOG_CATEGORIES | `"*"` | `app.py` + `newserver/log_manager.py` | 日志分类过滤，逗号分隔 |
| LOG_JSON | `"0"` | `app.py` + `newserver/log_manager.py` | `1` 输出 JSON 行日志 |
| LOG_BUFFER_SIZE | `"1000"` | `app.py` + `newserver/log_manager.py` | 内存日志缓冲上限 |
| LLM_PROVIDER | `"openai_compat"` | `newserver/agents/llm_action_provider.py` | LLM 提供方：`openai_compat/gemini` |
| LLM_TIMEOUT_SECONDS | `"60"` | `newserver/agents/llm_action_provider.py` | LLM 请求超时（秒） |
| LLM_MAX_RETRIES | `"2"` | `newserver/agents/llm_action_provider.py` | LLM 请求重试次数 |
| LLM_RETRY_BACKOFF_SECONDS | `"1.0"` | `newserver/agents/llm_action_provider.py` | LLM 重试退避基数 |
| LLM_FAILURE_THRESHOLD | `"3"` | `newserver/agents/llm_action_provider.py` | 连续失败达到阈值后进入冷却 |
| LLM_FAILURE_COOLDOWN_TICKS | `"60"` | `newserver/agents/llm_action_provider.py` | 失败冷却 tick 数，`0` 表示不冷却 |
| LLM_DEBUG_VIEW | `""` | `newserver/agents/llm_action_provider.py` | trace 下过滤 planner prompt 输出字段（逗号分隔） |
| LLM_PLANNER_MODEL | provider相关默认 | `newserver/agents/llm_action_provider.py` | Planner 模型名 |
| LLM_GROUNDER_MODEL | provider相关默认 | `newserver/agents/llm_action_provider.py` | Grounder 模型名 |
| LLM_BASE_URL | `"https://api.aabao.top"` | `newserver/agents/llm_action_provider.py` | OpenAI 兼容接口地址 |
| LLM_API_PREFIX | `"/v1"` | `newserver/agents/llm_action_provider.py` | OpenAI 兼容接口前缀 |
| LLM_API_KEY | `"REPLACE_ME"` | `newserver/agents/llm_action_provider.py` | OpenAI 兼容接口密钥 |
| GEMINI_BASE_URL | `"https://generativelanguage.googleapis.com"` | `newserver/agents/llm_action_provider.py` | Gemini 接口地址 |
| GEMINI_API_PREFIX | `"/v1beta"` | `newserver/agents/llm_action_provider.py` | Gemini 接口前缀 |
| GEMINI_API_KEY | `"REPLACE_ME"` | `newserver/agents/llm_action_provider.py` | Gemini 接口密钥 |
| LLM_FOCUS_AGENT_ID | `""` | `newserver/agents/llm_action_provider.py` | 聚焦日志的 agent id |
| LLM_FOCUS_LOG_PROMPTS | `"0"` | `newserver/agents/llm_action_provider.py` | 是否输出聚焦 prompt 日志 |

## 3. Checkpoint 机制

- 当前持久化机制是 checkpoint，`autosave` 已弃用。
- 启动恢复优先级：
  - 先看 `CHECKPOINT_RESTORE_FILE`
  - 未配置时看 `CHECKPOINT_RESTORE_DIR`（取最新 `tick_*.json`）
- 正常运行会按 `CHECKPOINT_EVERY_TICK` 在目录写入 `tick_000001.json` 这类文件。
- 若 `CHECKPOINT_INCLUDE_LOGS=1`，会一起写入 `event_log` 与 `interaction_log`。

## 4. 当前实现中的配置读取细节

- `app.py` 只读 `runtime_config.json`，不直接读系统环境变量。
- `build_default_llm_provider(...)` 会优先读取传入配置，不存在时再回退到系统环境变量。
- `LLM_FAILURE_THRESHOLD`、`LLM_FAILURE_COOLDOWN_TICKS`、`LLM_DEBUG_VIEW` 已接入统一配置链路，可直接写在 `runtime_config.json`。

## 5. 运行产物

- `checkpoints/`：checkpoint 文件目录，前端与恢复流程均基于该目录消费。

## 7. 维护约定

- 新增配置项时，必须同步更新：
  - 本文档
  - `runtime_config.example.json`
  - 对应代码默认值
- 如果配置语义发生变化，必须同时更新 checkpoint 与恢复相关说明。
